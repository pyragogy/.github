# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in any repository under the `pyragogy` GitHub organization, please report it privately by emailing:

**`info@pyragogy.org`**

Use a subject line that begins with `[SECURITY]`. Include:

- the affected repo and (if known) commit or version,
- a description of the issue and its impact,
- reproduction steps or a proof of concept where possible.

**Do not open public GitHub issues for security vulnerabilities.** Public disclosure before a fix is in place puts users of the project at risk.

## Response time

This is a small research-driven organization, so handling is best-effort. Expected timeline:

- **Acknowledgement** within 7 days of report.
- **Initial assessment** within 14 days.
- Coordinated disclosure timing agreed with the reporter.

## Scope

This policy covers all repositories owned by the `pyragogy` GitHub organization (https://github.com/pyragogy). Personal forks or repositories owned by individual contributors (e.g., under `FTG-003` or other accounts) are **out of scope** for this policy and should be reported to their respective maintainers.
